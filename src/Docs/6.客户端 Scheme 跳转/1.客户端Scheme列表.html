2. 重要参数说明


from，nm，chlid 这三个是公共参数，以上协议只要带了，都会解析并传递

2.1 from - 很重要的参数，请不要随便乱填，表明从哪里拉起的新闻

参数值 含义
weixin 从微信里的h5拉起
mobileQQPush 从手Q里的h5拉起
qqnews 从新闻内的h5拉起
static_shortcut O以上版本压力屏菜单拉起
other 协议没传的话，会默认补充other
baidu, youmeng. etc...等 其他的请填具体场景，比如从百度搜索页面跳过来

2.2 tab 页卡参数

参数值 含义
news_news 新闻
news_live 直播
news_recommend_main 推荐
user_center 我

2.3 iscomment （5525版本开始支持）

参数值 含义
0 跳转到正文页
1 跳转到评论页
2 跳转到评论页且打开评论输入框

2.4 articletype，文章类型，不知道可缺失，要填一定要是准确的

参数值 含义
0 普通文章
1 组图文章
100 专题文章
101 视频专题文章

2.5 jumpType，跳转类型，如果直接打开底层页可不填，如果打开类似动态详情页这种需填写，含义与articletyoe类似。

参数值 含义
0 普通文章
1 组图文章
100 专题文章
101 视频专题文章

2.6 topicId，话题id。如果文章属于一个话题，可以把话题 id传过来

2.7 act - 扩展host协议的接受类型，对于当前版本不包含在处理范围内的，会直接拉起主界面

参数值 含义
24hour 24小时必读
restore 拉起新闻回到前台(保持原样)
其他 拉起主界面

2.8 targetTabId - 会跳转到底层页指定的tab

hotPush: “推”页卡


3. 数据上报


3.1 各种拉起场景，有三个通用的拉起上报

渠道 事件ID
微信拉起的上报事件 boss_app_start_from_mm
QQ拉起的上报事件 boss_app_start_from_qq
其他拉起的上报事件 boss_app_start_from_others 该事件有个参数from，表征从哪里跳过来的， 也就是scheme协议中的from的值

3.2 jump_target：上报事件公共参数

参数值 含义
failed_or_cacel 跳转失败或者取消了
news_detail 通过getFullNews决定跳转
channel 二级频道
tab 页卡
news_search 搜索
news_game_recommend 个人中心里面的游戏
24hour 24小时页面
topic 事件聚合页
reader 读书二级页面
news_offline 新闻离线下载
hot_app_list 精品推荐页
tag tag页
focus_catagory 兴趣分类页
cp 企鹅号详情页
restore 恢复现场
redpacket 红包
reply_content_list 评论详情页
activity_page 活动页
upload_log 上传日志
speciallist 专题页

3.3 其他上报公共参数：

  1. startarticleid：

    外部拉起时的文章id，对应scheme中携带的 nm 字段值；本参数的存在周期为：从外部拉起开始，到app退到后台为止（退后台后该参数清空）。在此期间的所有数据上报，都会携带该参数。

  2. startarticletype：

    外部拉起的文章对应的articletype；存在周期同上

  3. startextras：

    外部拉起时scheme中携带的 startextras 字段值；存在周期同上


4. 升级注意事项


这里升级逻辑和之前的旧有升级弹窗复用一套逻辑，所以配置的信息要和旧有的弹窗保持一致，升级版本信息主要包括：

  • md5

  • message

  • version

  • url(目前包含了src和ostype)

以5.5.50为例：

qqnews://article_9528?act=app_download&md5=URLEncode的加密后的md5值&message=URLEncode后的message&version=5550&url=URLEncode后的url


URLEncode工具地址：http://splash.webdev.com/
